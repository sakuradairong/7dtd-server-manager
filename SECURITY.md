# 安全政策

## 支持版本

当前仅维护 `main` 分支上的最新版本。安全修复会优先合入 `main`。

## 报告安全问题

请不要在公开 issue 中披露可利用的安全细节。

如果你发现安全问题，请通过 GitHub 私密安全通报提交。
如果该入口不可用，请先创建一个不包含利用细节的普通 issue，
说明你需要私下联系维护者。

报告中请尽量包含：

- 受影响版本或提交
- 影响范围
- 复现步骤
- 期望行为和实际行为
- 你已验证的缓解方式（如有）

## 项目安全边界

- 本工具通过 Telnet 管理 7 Days to Die 专用服务器。
- Telnet 明文传输，不建议暴露到公网。
- 请仅在本地网络、受信任网络或 VPN 内使用。
- 保存的服务器配置可能包含明文密码；请保护好本机用户目录和备份文件。

## 维护者响应

维护者会尽力确认报告、评估影响并发布修复。由于这是社区项目，响应时间可能受维护者可用时间影响。
