# 排错手册

本文收集 7 Days to Die 服务器管理工具常见问题和排查步骤。

## 连接问题

### 症状：连接后立即断开或提示 "Invalid password"

1. 确认服务器已启用 Telnet，且端口、密码正确。
2. 检查服务器防火墙是否放行 Telnet 端口（默认 8081）。
3. 在"连接诊断"面板查看具体错误；点击"复制诊断"可粘贴到 issue 中。
4. 确认服务器控制台没有因多次失败而临时封禁你的 IP。

### 症状：长时间 "连接中..." 无响应

1. 检查服务器地址和端口是否可达：`telnet <host> <port>`。
2. 确认本地防火墙或安全软件未阻止应用出站连接。
3. 尝试增大连接超时时间（目前默认 10 秒，可在代码中调整 `DEFAULT_TIMEOUT_MS`）。

## 命令问题

### 症状：自定义命令无返回或提示超时

1. 部分命令（如 `listplayers`、`listents`、`getgamepref`）返回量大，需要更长的等待时间。
2. 检查命令是否包含换行符或多条命令；后端会拒绝多行命令以防止注入。
3. 查看"日志"面板的 `error` 级别输出，确认后端返回的具体错误。

### 症状：命令结果显示 "Not authenticated"

1. 表示 Telnet 密码认证失败，请断开并重新连接。
2. 如果服务器 Telnet 配置更改了密码，需要更新连接信息或保存的配置文件。

## 配置文件编辑

### 症状：选择 `serverconfig.xml` 后没有属性显示

1. 确认文件格式正确，根节点为 `<ServerSettings>`。
2. 当前 UI 仅显示一个固定的常用属性白名单；其他属性不会被列出，但也不会被删除。
3. 如果文件包含非法 XML，后端会拒绝加载并在日志中输出错误。

### 症状：保存配置后服务器不识别

1. 确认保存的是服务器实际使用的 `serverconfig.xml` 文件路径。
2. 部分属性需要重启服务器才能生效。
3. 检查文件编码是否为 UTF-8，后端会保留原始 XML 声明。

## 地图查看器

### 症状：地图目录选择后没有文件

1. 确认目录下存在图片文件（`png`、`jpg`、`jpeg`、`bmp`、`gif`、`webp`）。
2. 后端会对目录路径进行沙箱校验，确保是绝对路径且为目录。

### 症状：地图图片加载失败

1. 确认文件扩展名受支持。
2. 检查文件是否损坏或过大。
3. 使用地图预览的缩放/平移控件查看是否超出可视区域。

## 日志与导出

### 症状：日志不自动滚动

1. 当手动向上滚动时，自动滚动会自动暂停，并显示"恢复自动滚动"按钮。
2. 点击该按钮或滚动到底部可恢复自动滚动。

### 症状：导出日志失败

1. 确认选择了有效的保存路径。
2. 检查目标目录是否有写入权限。
3. 导出内容仅包含当前过滤后可见的日志条目。

## 构建与开发

### 症状：`npm run build` 失败

1. 确认已运行 `npm install`。
2. 确认 `src/common/constants.ts` 语法正确；构建会重新生成 `src/renderer/telnet-commands.gen.ts`。
3. 查看 TypeScript 错误输出并修复类型问题。

### 症状：`npm run test:rust` 失败

1. 确认已安装 Rust stable 工具链。
2. 运行 `cargo fmt --manifest-path src-tauri/Cargo.toml -- --check` 检查格式。
3. 运行 `cargo clippy --manifest-path src-tauri/Cargo.toml -- -D warnings` 检查警告。

## 仍无法解决？

请提交 GitHub issue 并提供以下信息：

- 操作系统和版本
- 应用版本或 `main` 分支提交哈希
- 复现步骤
- "连接诊断"或"日志"面板中的相关输出（请替换掉密码和敏感 IP）
