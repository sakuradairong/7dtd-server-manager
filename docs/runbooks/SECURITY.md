# 安全运行手册

本文面向维护者和高级用户，说明本项目的安全边界、已实施的防护措施以及安全事件响应流程。

## 威胁模型

本工具设计用于在受信任的网络环境中管理 7 Days to Die 专用服务器：

- **受信主体**：运行本工具的本地用户、被授权管理服务器的管理员。
- **不受信网络**：公网、未加密的 Wi-Fi、不受信任的局域网。
- **主要风险**：Telnet 明文传输、本地保存的密码/配置文件、通过文件路径参数可能导致的目录遍历。

## 已实施的防护措施

### Telnet 命令注入防护

- 后端拒绝包含换行符（`\n`、`\r`）的命令，防止单条命令拆分为多条命令。
- 所有命令在执行前都会经过 `validate_telnet_command` 校验。

### 密码处理

- 保存的配置文件密码使用 base64 混淆（`obf:` 前缀），不是加密，但可防止被轻易读取。
- 旧版本明文密码保持兼容，下次保存时会自动转为混淆格式。
- 连接诊断和日志中的密码会被替换为 `[password redacted]`。

### 文件路径沙箱

- `serverconfig.xml` 的加载和保存必须指向绝对路径且扩展名为 `.xml`。
- 地图图片读取必须指向绝对路径且扩展名为受支持的图片格式。
- 地图目录读取必须指向绝对路径且确实是目录。
- 路径中不允许包含 `..` 遍历组件。

### 内容安全策略（CSP）

- `tauri.conf.json` 中配置了严格的 CSP，限制脚本和样式来源。
- 前端不引入外部 CDN 资源。

## 安全事件响应

### 发现漏洞

1. 不要在公开 issue 中披露可利用细节。
2. 通过 GitHub 私密安全通报提交。
3. 提供复现步骤、影响版本、期望行为和已验证的缓解措施。

### 漏洞修复流程

1. 维护者在私密分支中修复漏洞。
2. 验证修复不会破坏现有功能（运行 `npm run check`）。
3. 发布新版 Release，并在发布说明中明确标注安全修复。
4. 根据漏洞严重程度和用户规模，决定是否为旧版本提供补丁或升级建议。

## 安全使用建议

- 仅在本地网络、受信任网络或 VPN 内使用 Telnet 连接。
- 不要与他人共享保存的配置文件；其中包含服务器密码。
- 定期更新到最新版本，以获取安全修复。
- 使用强密码并限制服务器 Telnet 端口的访问来源。

## 审计与合规

- 操作日志保存在本地应用数据目录的 `logs/` 文件夹中。
- 日志按日期滚动，保留最近 30 个文件。
- 审计时请注意日志中已自动替换密码字段，但其他敏感信息（如 IP）仍需手动脱敏。
